# Content Site Generator

將內容整理成結構化文件專案，同時產出 Markdown 檔案與可在瀏覽器直接開啟的靜態網站。

## 功能

- 根據使用者提供的內容（筆記、研究資料、文字稿等）自動規劃章節結構
- 產出 `docs/` 資料夾，內含各章節 Markdown 文件與目錄（INDEX.md）
- 產出靜態網站（`index.html`），無需 Web Server，直接用 `file://` 開啟
- 網站具備側邊欄導覽、全螢幕簡報模式、鍵盤快捷鍵
- 採多檔案架構（`sections/*.js`），每次修改只需讀寫對應章節檔，大幅節省 token

## 觸發時機

當使用者提到以下需求時使用此 skill：

- 建立文件網站、知識庫、靜態站台
- 把筆記或研究資料整理成可瀏覽的格式
- 製作「投影片風格」的 HTML 文件
- 提到「建站」、「靜態站」、「文件網站」、「資訊整理成網站」

## 產出結構

```
project/
├── CLAUDE.md          ← 專案說明與維護指引
├── template.html      ← CSS / HTML shell / JS 邏輯（~600 行，少動）
├── build.sh           ← 一鍵建置腳本
├── index.html         ← 建置輸出（瀏覽器開啟這個）
├── sections/
│   ├── slide-00.js    ← 封面 / 總覽
│   ├── slide-01.js    ← 第一章
│   └── ...
├── docs/
│   ├── INDEX.md       ← 目錄
│   ├── 01-slug.md     ← 各章節 Markdown
│   └── ...
└── assets/            ← 圖片（選用）
```

## 工作流程

1. **理解內容** — 分析使用者提供的資料，識別主題與閱讀順序
2. **規劃章節** — 列出章節大綱（封面 + 5–15 章 + 附錄），確認後再動工
3. **建立 docs/** — 先寫 INDEX.md，再逐一建立各章節 Markdown
4. **產出網站檔案** — 複製 template.html、建立 sections/*.js、複製 build.sh
5. **建置與驗證** — 執行 `./build.sh`，確認投影片數量與章節一致
6. **交付** — 完整資料夾給使用者，直接 `open index.html` 即可瀏覽

## 架構設計理念

採多檔案架構的核心原因：**迭代編輯時只需讀取對應章節檔（~100 行），而非整個 HTML（1500+ 行）**，大幅減少 AI 輔助修改時的 token 消耗。

## 語言慣例

文件內容預設使用**繁體中文**，技術術語與專有名詞可保留英文。
